# User Access & Permissions Lab

## Objective
Demonstrate Linux user account creation, permission enforcement, and controlled privilege escalation.

## Environment
- Platform: AWS EC2
- Operating System: Ubuntu Linux
- Access Method: SSH (Termius)

## Actions Performed
- Created a standard user account
- Verified default permission restrictions
- Granted sudo privileges using group membership
- Validated access changes through testing

## Key Takeaways
- Least privilege is enforced by default
- Elevated access should be granted intentionally
- User access should always be tested after changes
